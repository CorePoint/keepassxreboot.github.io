FAQ

Why KeePassXC instead of KeePassX?
You can find the motivation behind our fork on the Project page

Why KeePassXC instead of KeePass?
KeePass is a nice password manager that works fine but it's developed in C# and requires Microsoft's .NET framework.
Platforms beside Windows must use Wine or Mono instead, but this won't provide the native look and feel.
KeePassXC, on the other hand, is developed in C++ with the Qt library and it is cross-platform.

Is Auto-Type supported on OS X or Windows?
Yes, Auto-Type works cross-platform.

Which password database formats are compatible with KeePassXC?
KeePassXC currently uses the KeePass 2.x (.kdbx) password database format as its native format. It can also import KeePass 1.x (.kdb) databases.

Why do I get an error when I try to build from source for this platform?
Please follow every step on the Wiki:Install-Instruction-from-Source page

Why do the tray menu and in-app shortcuts not work on Ubuntu/Unity?
This is a bug caused by the appmenu-qt5 package.
You have 3 options:

  1. Remove the appmenu-qt5 package
  2. Set the environment variable UBUNTU_MENUPROXY=''
  3. Set the environment variable QT_QPA_PLATFORMTHEME=''
Note: When you choose the first or third option, KeePassXC will lose the Unity look&feel.
