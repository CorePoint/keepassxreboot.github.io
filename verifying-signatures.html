By verifying the signatures of KeePassXC releases you prove authenticity and integrity of the downloaded file.
This means that the file you just downloaded was created by the KeePassXC Team (the signature was made by one of the KeePassXC maintainers)
and that the contents of the file are exactly what the KeePassXC Team signed.

A more detailed article is available in the Qubes-OS project documentation.

Importing the Public Master Key

The KeePassXC public master key can be imported as follows

From a specific keyserver 
gpg --keyserver pool.sks-keyservers.net --recv-keys 0xBF5A669F2272CF4324C1FDA8CFB4C2166397D0D2
Manual download from here and import with gpg 
gpg --import ./keepassxc_master_signing_key.asc
Fetch via gpg 
gpg --fetch-keys https://keepassxc.org/keepassxc_master_signing_key.asc

These are the fingerprints of the master key and the current signing sub keys:

Verifying Releases

Every KeePassXC release is made of various files.
A .dmg image for OS X, a .zip archive with binaries for Windows and a self-contained .AppImage for GNU/Linux and other platforms. There is also a cross-platform .tar.bz2 source tarball.
Each of these files has a .sig sidecar file containing a detached signature created with one of the release sub keys.
Note: If you have trusted the master GPG key, the release sub keys are also trusted.
You can verify the authenticity and integrity of a downloaded package using its detached signatures by running the following command: 

$ gpg -v --verify KeePassXC-$VERSION-x86_64.AppImage.sig KeePassXC-$VERSION-x86_64.AppImage
gpg: Signature made Sun 22 Jan 2017 06:32:45 PM CET
gpg:                using RSA key 71D4673D73C7F83C17DAE6A2D8538E98A26FD9C4
gpg: Good signature from "KeePassXC Release <release 'AT` keepassxc ^DOT' org>" [unknown]
Additionally, there is a .DIGEST file that contains the SHA256 digest for each release file.
You can verify the integrity of a package by comparing its hash digest with the following command: 
$ sha256sum -c KeePassXC-$VERSION-x86_64.AppImage.DIGEST
KeePassXC-$VERSION-x86_64.AppImage: OK
