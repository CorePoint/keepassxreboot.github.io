--- layout: pages title: "The Project" permalink: project --- KeePassXC is a community fork of KeePassX which aims to incorporate stalled pull requests, features, and bug fixes that have never made it into the main KeePassX repository.

Additional Features
  • Autotype on all three major platforms (Linux, Windows, OS X)
  • Stand-alone password generator
  • Password strength meter
  • Use website's favicons as entry icons
  • Merging of databases
  • Automatic reload when the database changed on disk
For a full list of new features and changes, have a look at the full KeePassXC changelog.
What about KeePassHTTP?

KeePassXC does support KeePassHTTP for use with PassIFox in Firefox and chromeIPass in Google Chrome and Chromium. However, since the plugin in its current form offers a few pitfalls and we are very concerned about the safety of our users' passwords, we decided to not build it into the current main release. If need this feature, you can always download the source code and build it yourself with the CMake option -DWITH_XC_HTTP=On. We will reconsider adding the feature to the main release once we have fixed certain aspects.

Warning: If you decide to build KeePassXC yourself with KeePassHTTP support, please make sure that it only listens to localhost (127.0.0.1) and not to any network-routable interface and especially not to 0.0.0.0!

Requirements

KeePassXC has the following runtime requirements:

  • Qt5 (>= 5.2): qtbase and qttools5
  • libgcrypt (>= 1.6)
  • zlib
  • libmicrohttpd
  • libxi, libxtst, qtx11extras (optional for auto-type on X11/Linux)

For building KeePassXC from source code, the following tools have to exist in your PATH:

  • make
  • cmake (>= 2.8.12)
  • g++ (>= 4.7) or clang++ (>= 3.0)
